Robotics Competition Project

Sumo Bot

Programmed a competitive robot using C++, sensor-based detection, and control logic, earning 3rd place in competition.

View on GitHub Project Report

Objective

Build and program a robot capable of detecting opponents, staying inside the ring, and reacting quickly during a sumo-style robotics competition.



My Contribution

  • Programmed robot behavior in C++.
  • Implemented sensor-based opponent detection.
  • Tested and adjusted control behavior for competition performance.
  • Contributed to strategy and debugging before competition.

Technical Details

  • Programming language: C++
  • Sensor system: 8 Garmin LiDAR sensors arranged around the robot, with 5 along the front. Communicated using I2C
  • Control strategy: Detect which direction had an object within 15 inches, set target angle for PID, rotate until front end aligns and accelerate toward opponent.

Results

The robot placed 3rd in competition.

The robot's programming worked as expected given time constraints. System was reliable but provided too much time should the opponent choose to start their robot ahead of the mandatory 5 second delay.

The physical system allowed for many hits from opponents, but the 3D printed gears were at risk of shearing should the motors go full out. Due to the gearing system one side's electronic speed controller must be in reverse which is electronically limits itself to ~80% maximum throttle.

Had we chosen to make modifications after competition more of the gears would've been made from a carbon-fiber filament, along with proper gearing directions to allow for full use of the motors.
